# UI/UX 분석 및 개선 계획

## 1. 개요

본 문서는 RecipeMate 애플리케이션의 전반적인 사용자 인터페이스(UI) 및 사용자 경험(UX)을 분석하고, 일관성, 가독성, 사용성을 향상시키기 위한 개선 계획을 제안합니다. 분석은 제공된 코드베이스(`styles.css`, `base.html`, 각종 템플릿)와 디자인 가이드라인을 기반으로 수행되었습니다.

## 2. 전반적인 UI/UX 분석

### 긍정적인 점
- **통합된 레이아웃**: `layout/base.html`을 통해 전체 페이지의 구조가 일관되게 유지되고 있습니다.
- **Bootstrap 5 활용**: 최신 버전의 Bootstrap을 사용하여 반응형 디자인의 기반이 잘 마련되어 있습니다.
- **CSS 변수**: `styles.css`에 `--primary-color`와 같은 CSS 변수를 사용하여 주요 색상을 중앙에서 관리하고 있습니다.

### 개선 필요 사항
- **디자인 통일성 및 시각적 완성도 부족 (웹 디자이너 관점)**:
    - **조화롭지 않은 색상 조합**: 메인 색상인 주황색(`--primary-color`)은 잘 정의되어 있으나, 일부 요소에서 전체 테마와 충돌하는 색상이 사용되고 있습니다. 예를 들어, `index.html`의 이미지 플레이스홀더에 사용된 보라색/파란색 그라데이션(`linear-gradient(135deg, #667eea 0%, #764ba2 100%)`)은 주황색 테마와 조화롭지 않습니다.
    - **일관성 없는 컴포넌트 스타일**: '인기 공동구매' 섹션의 '모집중' 배지는 인라인 스타일로 파란색(` #2196F3`)이 적용되어, 다른 곳에서 사용되는 '모집중' 상태의 초록색(`bg-success`)과 다릅니다.
    - **부자연스러운 아이콘 사용**: 홈페이지의 '특별함' 섹션에서 사용된 이모지(🍳, 💰, 🤝)는 전문적인 느낌을 저해하며, 애플리케이션 전반에 사용되는 Bootstrap Icons와 통일성이 없습니다.
    - **시각적 계층 구조**: 홈페이지의 각 섹션이 연속적으로 배치되어 있어 시각적인 '숨 쉴 공간'이 부족합니다. 이로 인해 사용자가 콘텐츠를 구분하여 인지하기 어려울 수 있습니다.
- **색상 사용의 비일관성**:
    - **상태(Status) 색상**: 공동구매의 '모집 성공' 상태가 상세 페이지(`bg-info`, 파란색 계열)와 목록 페이지(`bg-success`, 초록색 계열)에서 다르게 표시됩니다. 사용자 요청에 따라 상세 페이지의 색상 기준으로 통일합니다.
    - **CSS 중복 정의**: `base.html`에 정의된 `<style>` 태그의 CSS 규칙이 `styles.css`의 내용과 중복됩니다. 이는 유지보수를 어렵게 하므로 `styles.css`로 통합해야 합니다.
- **컴포넌트 비일관성**:
    - **검색 폼**: 레시피 목록과 공동구매 목록의 검색 폼이 유사한 기능을 가졌음에도 불구하고 레이아웃, 필드 구성, 버튼 스타일이 다릅니다. 일관된 디자인으로 통일하여 사용자에게 예측 가능한 경험을 제공해야 합니다.
- **정보 과밀화**:
    - **공동구매 상세 페이지 사이드바**: 현재 사이드바는 구매 정보, 가격, 마감일, 배송 정보, 주최자 메뉴 등 너무 많은 정보가 한곳에 밀집되어 있어 가독성이 떨어집니다. 이를 논리적인 그룹으로 재구성하여 사용자가 정보를 쉽게 파악할 수 있도록 개선해야 합니다.
- **홈페이지 (`index.html`)**:
    - 전반적으로 깔끔하지만, '인기 레시피', '인기 공동구매' 등 각 섹션의 시각적 구분이 명확하지 않고, 일부 아이콘(이모지)이 전체적인 디자인 톤과 맞지 않습니다.

## 3. 개선 및 실행 계획

### 3.1. 홈페이지 UI 개선 (`index.html`)

- **목표**: 홈페이지의 가독성을 높이고 더 깔끔하고 전문적인 인상을 주도록 개선합니다.
- **실행 계획**:
    1.  **색상 및 테마 통일**:
        -   `index.html`의 이미지 플레이스홀더에 사용된 보라색/파란색 그라데이션을 제거합니다. 대신, `styles.css`에 `bg-light` 또는 회색 계열의 중립적인 색상을 사용한 플레이스홀더 스타일을 정의하여 적용합니다.
        -   '인기 공동구매' 섹션의 '모집중' 배지에서 인라인 스타일을 제거하고, 다른 곳과 동일하게 `bg-success` 클래스를 사용하도록 수정합니다.
    2.  **아이콘 교체**: 'RecipeMate의 특별함' 섹션의 이모지 아이콘을 전문적이고 일관성 있는 Bootstrap Icons (`bi-book-open`, `bi-tags`, `bi-people-fill` 등)으로 교체합니다.
    3.  **시각적 계층 구조 강화**:
        -   각 섹션 사이에 미묘한 상단 경계선(`border-top`)을 추가하거나, 섹션별 배경색(`bg-white`, `bg-light`)을 교차 적용하여 시각적 구분을 명확히 합니다.
        -   섹션 제목의 폰트 두께나 크기를 미세 조정하여 중요도를 강조합니다.
    4.  **코드 정리**: 페이지 전반의 불필요한 인라인 스타일을 제거하고 `styles.css`로 이전하여 유지보수성을 높입니다.

### 3.2. 검색 폼 UI 통일

- **목표**: `recipes/list.html`과 `group-purchases/list.html`의 검색 폼 디자인을 일관되게 만듭니다.
- **실행 계획**:
    1.  두 페이지의 검색 폼을 유사한 다중 열(multi-column) 레이아웃으로 재구성합니다.
    2.  필드(input, select)와 버튼의 수직 정렬을 맞추고, '검색' 버튼과 '필터 해제' 버튼의 스타일과 위치를 통일합니다.
    3.  카드(`div.card`) 내부에 폼을 배치하여 다른 콘텐츠와 시각적으로 분리합니다.

### 3.3. '모집성공' 상태 색상 통일

- **목표**: 공동구매 목록(`list.html`)의 '완료' 상태 배지 색상을 상세 페이지(`detail.html`)와 동일하게 수정합니다.
- **실행 계획**:
    1.  `group-purchases/list.html`에서 '완료'(`COMPLETED`) 상태를 나타내는 배지의 CSS 클래스를 `bg-success`에서 `bg-info`로 변경합니다.
        - **변경 전**: `<span th:if="${groupBuy.status.name() == 'COMPLETED'}" class="badge bg-success">완료</span>`
        - **변경 후**: `<span th:if="${groupBuy.status.name() == 'COMPLETED'}" class="badge bg-info">모집 성공</span>`
    2.  텍스트도 '완료'에서 '모집 성공'으로 변경하여 일관성을 맞춥니다.

### 3.4. 공동구매 상세 페이지 사이드바 재구성

- **목표**: 정보 과밀 상태인 사이드바의 가독성과 사용성을 높이기 위해 모던하고 직관적인 UI로 재설계합니다.
- **실행 계획**:
    1.  기존의 단일 정보 나열 방식에서 벗어나, 관련 정보를 그룹화한 여러 개의 카드(Card) 또는 아코디언(Accordion) 형태로 재구성합니다.
    2.  **구매 카드**: 가장 중요한 정보인 가격, 참여 인원, 핵심 액션 버튼('참여하기', '찜하기')을 포함하는 최상단 카드를 만듭니다.
    3.  **상세 정보 카드**: 마감일, 배송 방법, 직거래 장소 등 부가적인 정보를 별도의 카드로 분리합니다.
    4.  **주최자 메뉴 카드**: 주최자에게만 보이는 '공구 수정', '참여자 관리' 등의 버튼을 별도의 카드로 묶어 명확히 구분합니다.
    5.  각 카드에 아이콘과 명확한 헤더를 사용하여 정보 구분을 시각적으로 돕습니다.
    6.  반응형 디자인을 고려하여 모바일 화면에서도 정보가 깔끔하게 표시되도록 `floating-buttons` 클래스의 스타일을 유지 및 개선합니다.

## 4. 코드 스타일 및 유지보수 개선

- **목표**: 인라인 스타일을 제거하고 CSS를 중앙에서 관리하여 유지보수성을 높입니다.
- **실행 계획**:
    1.  `index.html`, `group-purchases/detail.html`, `base.html` 등 템플릿 파일에 존재하는 모든 `<style>` 블록과 인라인 `style="..."` 속성을 `styles.css` 파일로 이전합니다.
    2.  중복되는 CSS 규칙을 병합하고, CSS 변수를 적극적으로 활용하여 코드의 재사용성을 높입니다.
