api가 제공하는 재료와 용량을 파싱하는 거의 초점
- 이상하게 파싱되는 걸 완전히 없애기 보단, 최소화시키는 쪽으로
- 풍부한 정보 전부 제공하는 쪽보단, 일부 누락되는 걸(물론 너무 누락되면 안 됨) 감안하더라도 나머지 정상적인 파싱된 것들을 살리는 쪽으로
    - 즉 예시로 "연두부 75g(3/4모)" 이렇게 제공되면. 재료는 연두부. 용량이 75g(3/4모) 이렇게 파싱되는 게 기본인데
    - 복잡한 특수문자나 특이 케이스. 즉 꽤나 이상하게 되는 파싱들은. 그냥 부가적인 건 버리고 핵심만 남겨도 될 듯. 다 살리려고 하지 말고.
    - 만약 "곁들이 야채 : " 이런 식으로 부가적인 정보는 아예 파싱에서 빼버리든지. 물론 이거 억지로 빼려다가 더 꼬이고 복잡해질 거 같으면 하지 말고.
- 이상하게 파싱되더라도 사용자가 "레시피 기반 공구 만들기"에서 수정 및 삭제 가능하도록 하면 되니까.
