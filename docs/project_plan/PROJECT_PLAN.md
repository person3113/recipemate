## 🔌 핵심 API 선정 (3개)

### ✅ 1. TheMealDB API (필수 - 레시피 데이터)

**용도**: 레시피 검색, 상세 정보, 재료 리스트 제공
**비용**: 개발용 무료 (키="1")
**활용 기능**:
- 레시피 검색 (`search.php?s=이름`)
- 레시피 상세 조회 (`lookup.php?i=ID`)
- 재료별 레시피 필터 (`filter.php?i=재료`)
- 랜덤 레시피 추천 (`random.php`)
- 카테고리별 조회 (`categories.php`)

**데이터 구조**:

- `strMeal` (레시피명)
- `strIngredient1~20` (재료 목록)
- `strMeasure1~20` (분량)
- `strInstructions` (조리법)
- `strMealThumb` (이미지)
- `strCategory` (카테고리)

### ✅ 2. 식품안전나라 COOKRCP01 API (준필수 - 한국 레시피)

**용도**: 한국 음식 레시피 + 영양 정보
**비용**: 무료 (인증키 발급 필요)
**활용 기능**:
- 한식 레시피 700+ 제공
- 영양 정보 (칼로리, 단백질, 탄수화물 등)
- 조리 단계별 이미지
- 레시피명, 재료 정보 제공

**장점**: TheMealDB(영어)를 보완, 한국 사용자 경험 강화

### 🔄 3. Google Cloud Translation API (선택 - 번역)
**용도**: TheMealDB 영어 데이터 → 한국어 자동 번역
**비용**: 월 50만 문자 무료
**활용 시점**: 5주차 이후 여유 있으면 추가

## 🎯 10. 주요 사용자 시나리오 (전체 흐름)

### 시나리오 1: 신규 사용자 - 레시피 보고 공구 참여

```
1. 홈 접속 (비로그인)
   ↓
2. 인기 레시피 캐러셀에서 "까르보나라" 클릭
   ↓
3. 레시피 상세 확인 (재료, 조리법)
   ↓
4. "진행 중인 공구" 섹션에서 공구 발견
   ↓
5. [공구 상세 보기] 클릭
   ↓
6. 공구 상세 확인 (가격, 수령 방법, 주최자 정보)
   ↓
7. [참여하기] 클릭
   ↓
8. 로그인 페이지로 리다이렉트
   ↓
9. [회원가입] 클릭 → 회원가입 완료
   ↓
10. 자동 로그인 → 공구 상세로 돌아옴
   ↓
11. [참여하기] 다시 클릭
   ↓
12. 수령 방법 선택 (직거래/택배)
   ↓
13. [확인] → 참여 완료!
   ↓
14. 마이페이지 > 참여 공구에서 확인 가능

```

### 시나리오 2: 기존 사용자 - 레시피 기반 공구 만들기

```
1. 로그인 상태에서 [레시피 탐색] 메뉴 클릭
   ↓
2. 검색창에 "카레" 입력 → Enter
   ↓
3. 검색 결과에서 "일본식 카레" 클릭
   ↓
4. 레시피 상세에서 재료 확인
   ↓
5. 인원 설정: 4명 → 10명으로 변경
   ↓
6. 이미 있는 재료 체크 해제 (예: 쌀, 소금)
   ↓
7. [이 재료로 공구 만들기] 버튼 클릭
   ↓
8. 공구 만들기 페이지로 이동 (자동 입력됨)
   - 제목: "일본식 카레 재료 공동구매"
   - 내용: 재료 목록 자동 나열
   - 이미지: 레시피 이미지 첨부
   ↓
9. 추가 정보 입력:
   - 총 금액: 80,000원
   - 목표 인원: 10명 (1인당 8,000원 자동 계산)
   - 마감일: 다음주 금요일 선택
   - 수령 방법: 직거래 + 택배 선택
   - 만남 장소: "서울대 정문" 입력
   - 택배비: 3,000원 입력
   ↓
10. [미리보기] → 확인 → [등록하기]
   ↓
11. 공구 등록 완료! 공구 상세로 이동
   ↓
12. 마이페이지 > 내 공구에서 확인

```

### 시나리오 3: 공구 완료 후 후기 작성

```
1. 마이페이지 > [참여 공구] 탭
   ↓
2. 완료된 공구 확인 (상태: 완료 ✅)
   ↓
3. [후기 작성] 버튼 클릭
   ↓
4. 후기 작성 팝업:
   - 별점: ⭐⭐⭐⭐⭐ (5점) 선택
   - 텍스트: "재료가 신선했고 주최자가 친절했습니다!" 입력
   ↓
5. [등록] 클릭
   ↓
6. 후기 저장 완료
   - 주최자 매너온도: 36.5°C → 37.0°C (+0.5)
   - 내 포인트: +5P
   - 주최자에게 알림 발송
   ↓
7. 커뮤니티 > [레시피 후기]에도 자동 게시 (선택 기능)

```

### 시나리오 4: 커뮤니티 활동

```
1. 네비게이션 > [커뮤니티] 클릭
   ↓
2. [꿀팁 공유] 탭 선택
   ↓
3. [글쓰기] 버튼 클릭
   ↓
4. 글쓰기 페이지:
   - 카테고리: 꿀팁 공유 선택
   - 제목: "양파 오래 보관하는 법" 입력
   - 내용: 꿀팁 작성
   - 이미지: 사진 3장 업로드
   - 태그: #식재료보관 #양파 추가
   ↓
5. [등록하기]
   ↓
6. 게시글 등록 완료! 포인트 +3P
   ↓
7. 다른 사용자들이 댓글 작성
   ↓
8. 알림 수신: "OOO님이 댓글을 남겼습니다"
   ↓
9. [알림] 탭에서 확인 → 댓글 클릭 → 게시글로 이동
   ↓
10. 댓글에 [답글] 작성

```

## 📝 13. API 연동 상세

### TheMealDB API 주요 엔드포인트

```jsx
// 1. 레시피 검색 (이름)
GET <https://www.themealdb.com/api/json/v1/1/search.php?s=파스타>
→ 응답: { meals: [ {...레시피 객체...} ] }

// 2. 레시피 상세 (ID)
GET <https://www.themealdb.com/api/json/v1/1/lookup.php?i=52772>
→ 응답: { meals: [ {...상세 레시피...} ] }

// 3. 재료별 레시피 필터
GET <https://www.themealdb.com/api/json/v1/1/filter.php?i=chicken>
→ 응답: { meals: [ {...레시피 간단 정보...} ] }

// 4. 랜덤 레시피
GET <https://www.themealdb.com/api/json/v1/1/random.php>
→ 응답: { meals: [ {...랜덤 레시피...} ] }

// 5. 카테고리 목록
GET <https://www.themealdb.com/api/json/v1/1/categories.php>
→ 응답: { categories: [ {...카테고리 목록...} ] }

// 6. 이미지 URL 규칙
레시피 이미지: strMealThumb (예: /images/media/meals/llcbn01574260722.jpg)
재료 이미지: <https://www.themealdb.com/images/ingredients/{재료명}.png>

```

### 식품안전나라 API

```jsx
// 레시피 조회
GET <http://openapi.foodsafetykorea.go.kr/api/{인증키}/COOKRCP01/json/1/100>
→ 응답: { COOKRCP01: { row: [ {...레시피 배열...} ] } }

// 주요 필드:
RCP_NM: 메뉴명
RCP_PARTS_DTLS: 재료정보
MANUAL01~20: 조리 단계 설명
MANUAL_IMG01~20: 조리 단계 이미지
INFO_ENG: 열량
INFO_CAR: 탄수화물
INFO_PRO: 단백질
INFO_FAT: 지방
INFO_NA: 나트륨

```

## "레시피메이트" 기획서 - 보완 및 제안 사항 (Areas for Improvement)

기획서가 이미 매우 훌륭하지만, 실제 개발 및 운영 관점에서 다음 몇 가지를 추가로 고려하면 더욱 완성도를 높일 수 있습니다.

2. **가격 책정의 현실성 문제**:
    - '레시피 기반 공구 만들기' 시 사용자가 직접 '총 금액'을 입력하게 되어 있습니다. 하지만 다양한 재료의 현재 시장 가격을 개인이 정확히 알기는 어렵습니다.
    - **제안**: 초기 버전에서는 현재 방식(사용자 직접 입력)을 유지하되, **'보너스 아이디어'**나 **'향후 개선 과제'**로 **'주요 식재료 평균 가격 정보 제공'** 기능을 추가하면 좋겠습니다. (예: 별도의 크롤링이나 공공데이터 API를 활용) 이는 사용자가 더 합리적인 가격으로 공구를 설정하는 데 도움을 줄 것입니다.
3. **'매너온도' 시스템의 구체화**:
    - 현재 후기 별점에 따라 매너온도가 변동되는 로직은 좋습니다. 다만, 분쟁 발생 시나 비정상적인 평가(일명 '평점 테러')에 대한 대응책이 필요합니다.
    - **제안**: **'신고 기능'**을 추가하고, '누적 신고 횟수에 따른 매너온도 차감' 로직을 명시하면 더 신뢰도 높은 시스템을 구축할 수 있습니다. 마이페이지 설정 탭의 '계정 탈퇴'처럼, 운영 정책을 조금 더 구체화하는 것이 좋습니다.
